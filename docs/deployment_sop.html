Script TF 容器化部署 SOP

版本 1.0更新日期 2026-06-24

目的：將 Script TF（跨品牌機器人劇本轉換工具）打包成 Docker image，部署到另一台 Ubuntu PC 上運行。目標機器的 Ollama 跑在 host（非容器），提供進階 AI 對話功能。

0. 前置準備

項目說明
來源機器已安裝 Docker（用於建置 image），本機 Windows 即可
目標機器Ubuntu 20.04+，已安裝 Docker CE
傳輸方式USB / SCP / 內部檔案伺服器（假設兩機無法直接 push 到同一個 registry）
Ollama目標機器上需自行安裝並啟動，容器不含 Ollama

1. 在來源機器建置 Image

專案根目錄已包含 Dockerfile 與 .dockerignore。在專案目錄下執行：

docker build -t script-tf:latest .

建置完成後可驗證：

docker images | grep script-tf

2. 匯出 Image 成檔案

沒有共用 registry 時，用 docker save 將 image 打包成單一檔案：

docker save script-tf:latest -o script-tf.tar

壓縮以加快傳輸（選用）：

gzip script-tf.tar   # 產生 script-tf.tar.gz
若兩機都能存取同一個內部 registry，更建議用 docker push / pull，可省略 save/load 步驟。

3. 傳輸到目標 Ubuntu 機器

用 SCP（建議，假設可 SSH 連線）：

scp script-tf.tar.gz user@<ubuntu-ip>:/home/user/

或用 USB 隨身碟複製過去，再插到目標機器上。

4. 目標機器安裝 Docker CE（若尚未安裝）

# 移除舊版（若有）
sudo apt-get remove docker docker-engine docker.io containerd runc

# 安裝必要套件
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg

# 加入 Docker 官方 GPG key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 加入 Docker repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 驗證
sudo docker --version

（選用）讓目前使用者免 sudo 執行 docker：

sudo usermod -aG docker $USER
newgrp docker

5. 載入 Image

gunzip script-tf.tar.gz          # 若有壓縮，先解壓
docker load -i script-tf.tar
docker images | grep script-tf   # 確認載入成功

6. 確認 Host 上的 Ollama

# 安裝（若尚未安裝）
curl -fsSL https://ollama.com/install.sh | sh

# 下載模型
ollama pull llama3.2:3b

# 啟動服務（Linux 通常會註冊為 systemd service，自動啟動）
sudo systemctl status ollama
# 若未啟動：
sudo systemctl enable --now ollama

# 驗證
curl http://localhost:11434
# 應回應 "Ollama is running"
容器需要透過 host.docker.internal 連到 host 上的 Ollama。Linux 上預設不支援這個 DNS 名稱，啟動容器時必須加上 --add-host=host.docker.internal:host-gateway（見下一步）。

7. 啟動容器

docker run -d \
  --name script-tf \
  --restart unless-stopped \
  -p 7860:7860 \
  --add-host=host.docker.internal:host-gateway \
  script-tf:latest
參數用途
-d背景執行
--restart unless-stopped機器重開後自動啟動容器
-p 7860:7860對外開放 Web UI 連接埠
--add-host=host.docker.internal:host-gateway讓容器能解析到 host IP，連線 Ollama

若要自訂 Ollama 模型或位址，可加上環境變數：

docker run -d \
  --name script-tf \
  --restart unless-stopped \
  -p 7860:7860 \
  --add-host=host.docker.internal:host-gateway \
  -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
  -e OLLAMA_MODEL=llama3.2:3b \
  script-tf:latest

8. 驗證部署

# 確認容器運行中
docker ps

# 查看 log
docker logs -f script-tf

# 瀏覽器開啟（從目標機器或同網段其他電腦）
http://<ubuntu-ip>:7860

進入 Web UI 後，於右欄對話框測試自然語言指令，確認能連到 Ollama（若回應正常代表容器 → host 的連線成功）。

9. 更新部署（後續版本）

# 在來源機器重新 build + save，傳輸新的 tar 後，於目標機器：
docker stop script-tf && docker rm script-tf
docker load -i script-tf-new.tar
docker run -d --name script-tf --restart unless-stopped -p 7860:7860 \
  --add-host=host.docker.internal:host-gateway script-tf:latest

10. 疑難排解

問題排查方向
瀏覽器無法開啟 7860檢查 docker ps 容器是否在跑；檢查 Ubuntu 防火牆 sudo ufw status，需開放 7860
AI 對話功能無回應 / 報錯確認 curl http://localhost:11434 在 host 上有回應；確認容器內可解析 host.docker.internal（docker exec -it script-tf getent hosts host.docker.internal）
容器啟動後立即退出docker logs script-tf 查看錯誤訊息，常見為依賴安裝失敗或 port 衝突
Ollama 連線被拒絕確認 Ollama 監聽位址未被限制為僅 127.0.0.1（預設行為對容器是足夠的，因為走 host-gateway），必要時設定 OLLAMA_HOST=0.0.0.0 並重啟服務

附錄：常用指令速查

# 進入容器內部 shell
docker exec -it script-tf bash

# 停止 / 啟動 / 重啟容器
docker stop script-tf
docker start script-tf
docker restart script-tf

# 移除容器與 image（重新部署前清理）
docker rm -f script-tf
docker rmi script-tf:latest
